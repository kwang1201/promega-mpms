## 1. 배경 및 문제 정의

### 1.1 배경

Promega Korea(PK) 마케팅 팀은 연간 다수의 학회/전시에 참가하며, 부스 설계부터 브로셔류 인쇄물, 배너, 판촉물 등 다양한 마케팅 자료를 외부 에이전시와 협업하여 제작하고 있다. 현재 이 과정은 카카오톡, 이메일, 공유폴더 등 분산된 채널을 통해 진행되고 있어 체계적인 관리가 어려운 상황이다.

### 1.2 현재 문제점 (Pain Points)

| # | 문제 영역 | 상세 설명 |
|---|----------|----------|
| 1 | **소통 채널 분산** | 외주업체와 카카오톡으로 파일 주고받고, 공유폴더에 다운받아 리뷰하는 비효율적 프로세스 |
| 2 | **리뷰 이력 추적 불가** | N차 리뷰가 반복되지만 버전별 변경 이력과 피드백 내역이 체계적으로 관리되지 않음 |
| 3 | **진행 현황 불투명** | 병렬 트랙(부스/배너/판촉물/포스터/설문)의 전체 진행 상황을 한눈에 파악하기 어려움 |
| 4 | **비용 관리 분산** | 견적서/계산서가 별도 엑셀로 관리되어 기안 연동과 정산 추적이 번거로움 |
| 5 | **알림 부재** | 단계 전환 시 관련자에게 수동으로 연락해야 하며, 누락 발생 가능성 높음 |

## 2. 시스템 개요

### 2.1 목적

학회 오너가 MS(Marketing Service)에 제작을 요청하는 시점부터 외주 제작, 리뷰, 승인, 납품, 정산까지의 전과정을 통합 관리하는 시스템을 구축한다.

> **핵심 범위:** 학회 오너 → MS Request 시점 ~ 학회 후 정산 완료까지
> (참석 기안 작성/승인은 시스템 외부 선행 프로세스)

### 2.2 핵심 가치

| 가치 | 설명 |
|------|------|
| **투명성** | 병렬 트랙 전체 진행 상황을 학회 오너/MS/외주업체 모두 실시간 확인 |
| **추적성** | N차 리뷰 이력, 파일 버전, 피드백 내역을 완전히 보존 |
| **효율성** | 카톡 대신 시스템에서 파일 교환 및 리뷰, 자동 알림으로 수동 연락 제거 |
| **비용 통제** | 견적→기안→제작→계산서→정산 전 비용 흐름을 시스템 내에서 통합 관리 |

## 3. 현행 프로세스 분석 (As-Is)

현재 학회 전시 프로세스는 6단계로 구성되며, 승인 후 제작 단계에서 병렬 트랙(A~E)이 동시 진행된다.

### 3.1 전체 프로세스 단계

| 단계 | 프로세스 | 주요 업무 | 담당 |
|------|---------|----------|------|
| ① | 참석 기안 작성/승인 | 학회 참석 기획안 작성 및 승인 | 학회 오너 |
| ② | 기획 미팅 | 목적 공유, 부스 운영 프로그램 논의, KPI 설정 | 오너/PM/MS |
| ③ | 전시 기안 준비 | 부스 디자인, 견적 요청, 예산서 작성 | MS |
| **④** | **승인 후 제작** | **A. 부스 백월/디자인 · B. 배너 · C. 판촉물 · D. SCM 포스터 · E. 설문/테이블텐트** | **오너/MS/PM/외주** |
| ⑤ | 학회 전 최종 준비 | 물품 체크, 포장/출고, 브리핑, 부스 검수 | 오너/MS/외주 |
| ⑥ | 학회 후 보고/정산 | 리드 정리, CRM 업로드, KPI 평가, 비용 정산 | 오너/MS |

> **시스템 관리 범위:** ③ 전시 기안 준비 단계부터 ⑥ 학회 후 보고/정산까지
> (①②는 시스템 외부 선행 프로세스)

### 3.2 ④ 승인 후 제작 — 병렬 트랙 상세

#### 트랙 A: 부스 백월 · 디자인

| 세부 업무 | 담당자 | 비고 |
|----------|--------|------|
| 부스 백월 메시지·키워드 작성 | 학회 오너 | |
| 부스 디자인 작업 | 외주업체 | MS 관리 |

#### 트랙 B: 배너

| 세부 업무 | 담당자 | 비고 |
|----------|--------|------|
| 배너 콘텐츠 작성 | PM | |
| 배너 디자인 기획 | MS | |
| 배너 디자인 작업 | 외주업체 | MS 관리 |
| 배너 인쇄 | MS | |

#### 트랙 C: 판촉물

| 세부 업무 | 담당자 | 비고 |
|----------|--------|------|
| 판촉물(Giveaway) 시안 확인 및 제작 | MS | |

#### 트랙 D: SCM 포스터

| 세부 업무 | 담당자 | 비고 |
|----------|--------|------|
| SCM 포스터 콘텐츠 작성 | SCM | |
| SCM 포스터 인쇄 | MS | |

#### 트랙 E: 설문 · 테이블텐트

| 세부 업무 | 담당자 | 비고 |
|----------|--------|------|
| 설문 내용 초안 작성 | 학회 오너 | |
| 설문 프로그램 Form 생성 | MS | |
| 테이블 텐트 작업 | MS | |

### 3.3 현행 외주 업무 흐름 (인쇄물 기준, MS 인터뷰 기반)

1. 학회 오너가 MS에 **초안 전달** 및 견적 요청
2. MS가 외주업체에 **견적 요청** 및 수령
3. 견적 기반으로 **기안 승인** (전시 기안)
4. MS가 외주업체와 **제작 진행** (카카오톡으로 파일 교환)
5. 외주업체 → 카톡으로 파일 전달 → **공유폴더 다운로드** → 오너+MS 공동 리뷰
6. **리뷰 N차 반복** (피드백 → 수정 → 재리뷰)
7. 완료 후 **계산서 수령** → 기안자에게 전달 → 기안 첨부

## 4. 목표 프로세스 (To-Be)

시스템 도입 후 프로세스 변화:

| 영역 | As-Is (현행) | To-Be (목표) |
|------|-------------|-------------|
| 요청 접수 | 구두/이메일/카톡 혼재 | 시스템에서 표준화된 요청 폼 |
| 파일 교환 | 카카오톡 → 공유폴더 수동 저장 | 시스템 업로드 + 자동 버전 관리 |
| 리뷰 프로세스 | 카톡 코멘트, 구두 피드백 | 인라인 마크업 + 스레드 코멘트 |
| 진행 현황 | 개별 확인 필요 | 대시보드에서 병렬 트랙 한눈에 파악 |
| 비용 관리 | 별도 엑셀 | 견적서/계산서 첨부 + 자동 정산 추적 |
| 알림 | 수동 연락 | 단계별 자동 알림 (Email/Slack/시스템) |
| 외주업체 참여 | 카톡으로만 소통 | 전용 포털에서 직접 파일 업로드/상태 업데이트 |

## 5. 핵심 기능 모듈

### 5.1 프로젝트 관리

학회별 프로젝트를 생성하고 전체 진행 상황을 시각적으로 관리한다.

- **학회별 프로젝트 카드:** 학회명, 일정, 장소, 학회 오너, 예산
- **Kanban 보드:** 기안준비 → 견적진행 → 제작중 → 리뷰중 → 최종준비 → 학회완료 → 정산완료
- **병렬 트랙 뷰:** 부스/배너/판촉물/포스터/설문 각 트랙의 진행율 시각화
- **우선순위/마감일 기반 자동 정렬** 및 지연 경고

### 5.2 파일 및 버전 관리

카카오톡 파일 교환을 대체하는 핵심 기능이다.

- **외주업체가 시스템에 직접 파일 업로드** (카톡 대체)
- **자동 버전 번호 부여:** v1 → v2 → v3 (리뷰 회차와 연동)
- **파일별 변경 이력** 및 업로더 정보 자동 기록
- **드래그앤드롭 업로드:** AI, PSD, PDF, InDesign 등
- **Promega 브랜드 에셋 라이브러리** (로고, 컬러, 템플릿)
- **견적서/계산서 PDF 첨부** 및 기안 연동

### 5.3 리뷰 및 피드백

N차 리뷰 프로세스를 체계적으로 관리한다.

- **파일별 인라인 마크업 및 코멘트** (오너+MS 공동 리뷰)
- **리뷰 회차 자동 추적** (1차 리뷰, 2차 리뷰, ...)
- **리뷰 완료 시 외주업체에 자동 알림** + 수정 요청 사항 요약
- **승인/반려/조건부 승인** 상태 관리
- **감사 로그:** 누가, 언제, 무엇을 승인/반려했는지

### 5.4 비용 관리

견적부터 정산까지의 비용 흐름을 통합 관리한다.

- **견적서 첨부** 및 금액 등록
- **기안 승인 상태 연동** (견적 승인 → 제작 시작 허용)
- **제작 완료 후 계산서 업로드** 및 기안자 연동 알림
- **학회별 총 비용 집계** 및 예산 대비 실적 현황
- **월별/분기별 지출 추이** 리포트

### 5.5 대시보드

MS 및 관리자를 위한 실시간 현황 및 분석 기능.

- 진행 중/완료/지연 프로젝트 현황 요약
- 학회별 병렬 트랙 진행률 시각화
- 외주업체별 성과: 납기 준수율, 평균 리뷰 횟수, 평균 소요일
- 예산 현황: 학회별/월별 지출 현황
- 자동 월간 리포트 생성

## 6. 알림 시스템 (Notification)

각 단계 전환 및 주요 이벤트 발생 시 관련자에게 자동 알림을 발송한다.

| 이벤트 | 수신자 | 채널 | 타이밍 |
|--------|--------|------|--------|
| 새 요청 등록 | MS 담당자 | Email + 시스템 | 즉시 |
| 견적서 업로드 | 학회 오너 (기안자) | Email + 시스템 | 즉시 |
| 기안 승인 완료 | MS + 외주업체 | Email + 시스템 | 즉시 |
| 외주업체 파일 업로드 | 오너 + MS (리뷰자) | Email + 시스템 | 즉시 |
| 리뷰 완료 / 피드백 전달 | 외주업체 | Email + 시스템 | 즉시 |
| 최종 승인 | MS + 외주업체 | Email + 시스템 | 즉시 |
| 계산서 업로드 | 학회 오너 (기안자) | Email + 시스템 | 즉시 |
| 마감일 3일 전 경고 | 담당자 전체 | Email | 자동 |
| 마감일 초과 (Overdue) | 담당자 + 관리자 | Email + 시스템 | 자동 (매일) |
| 주간 진행 요약 | MS 관리자 | Email | 매주 월요일 |

알림 채널은 Email을 기본으로 하며, 추후 Slack 연동을 확장할 수 있다. 시스템 내 알림 센터에서 전체 이력을 확인할 수 있다.

## 7. 사용자 역할 및 권한

RBAC(Role-Based Access Control) 기반으로 역할별 접근 권한을 제어한다.

| 역할 | 설명 | 주요 권한 | 예시 담당자 |
|------|------|----------|-----------|
| **학회 오너** | 학회 참석 기획 및 결과 보고 | 요청 생성, 콘텐츠 제공, 리뷰, KPI 보고 | PM / 학회 담당자 |
| **MS 담당자** | 전시 실무 총괄 운영 | 전체 관리, 외주 조율, 승인, 정산 | Mikyoung Kim |
| **MS 관리자** | 마케팅 서비스 팀 리더 | 최종 승인, 리포트, 예산 관리 | MS Manager |
| **외주업체** | 외부 디자인/인쇄 업체 | 할당된 프로젝트 열람, 파일 업로드, 코멘트 | 디자인 에이전시 |
| **SCM** | 과학 커뮤니케이션 | 포스터 콘텐츠 제공 | SCM Manager |

| **notifications** | id, user_id, event_type, message, read, created_at | 알림 이력 및 읽음 상태 |
| **activity_log** | id, project_id, user_id, action, details, created_at | 모든 변경 이력 감사 로그 |
| **agencies** | id, name, contact_person, email, phone, specialty | 외주업체 정보 |
| **users** | id, name, email, role, agency_id | 사용자 (역할, 소속) |
